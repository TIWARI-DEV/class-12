Database Concepts

Past Year Questions of DBMS (2020-2024 and Model Questions)

1
What is Database and DBMS? List out the advantages and disadvantages of DBMS.

A database is an organized collection of related data stored electronically in a computer system. For example, a school might have a database containing student records, course information, and grades.

A Database Management System (DBMS) is software that allows users to interact with databases. Popular DBMS include MySQL, Oracle, and Microsoft SQL Server. It provides tools for creating, modifying, and querying databases.

Advantages of DBMS

Data redundancy is minimized, data integrity is maintained through constraints, data can be shared among multiple users concurrently, and security features protect sensitive information.

Disadvantages of DBMS

Setting up a DBMS requires significant investment in hardware and software. Complex systems require specialized personnel to manage them, and converting from traditional systems can be challenging.

2
Differentiate between file processing system and DBMS. Give at least four points.

File processing systems store data in separate files for each application, while DBMS stores data in a centralized repository accessible to multiple applications.

In file processing systems, data redundancy is common as the same information may be stored in multiple files. DBMS minimizes redundancy by storing data in a structured format.

File systems have limited data sharing capabilities and often require data duplication. DBMS allows concurrent access to data by multiple users without compromising integrity.

DBMS provides robust security features like user authentication and access control, which are typically lacking in file processing systems.

Example

In a file system, student information might be duplicated in separate files for registration, grades, and library. In DBMS, this information is stored once and accessed by all departments.

3
Explain the different models of DBMS with advantages and disadvantages.

Hierarchical Model: Organizes data in a tree-like structure with parent-child relationships. Used in early banking systems. Advantages include efficiency for 1:N relationships. Disadvantages include complexity in representing M:N relationships.

Network Model: Similar to hierarchical but allows multiple parent-child relationships. Used in airline reservation systems. Advantages include efficient complex queries. Disadvantages include complexity in design and maintenance.

Relational Model: Stores data in tables (relations) with rows and columns. Used in most modern systems. Advantages include simplicity and flexibility. Disadvantages include performance issues with very large databases.

Object-Oriented Model: Stores data as objects with attributes and methods. Used in CAD systems. Advantages include handling complex data. Disadvantages include lack of standardization.

Example

In a relational model, student data would be in one table and courses in another, linked by keys. In hierarchical, students might be parents to their enrolled courses.

4
What is a relational database? How is it different from other database models?

A relational database organizes data into tables (relations) consisting of rows (tuples) and columns (attributes). Tables are related through keys (primary and foreign keys).

Unlike hierarchical and network models that use pointers and complex structures, relational databases use simple tabular structures that are easy to understand and manipulate.

Relational databases use SQL (Structured Query Language) for data manipulation, which is standardized across different systems. Other models typically use proprietary query languages.

Example

In a library system, a relational database would have separate tables for books, authors, and borrowers linked by keys. A hierarchical model might represent books as children of authors, making it harder to represent books with multiple authors.

5
What is data redundancy? How does DBMS help in reducing it?

Data redundancy occurs when the same piece of data is stored in multiple places within a database or across multiple systems. This wastes storage space and can lead to data inconsistency.

DBMS reduces redundancy through normalization, which organizes data to minimize duplication. By using relationships between tables, data is stored only once and referenced where needed.

Foreign keys in relational databases allow linking to data in other tables instead of duplicating it. Constraints ensure that updates propagate to all references.

Example

Without DBMS, student addresses might be stored in registration, library, and billing files. With DBMS, the address is stored once in a student table and referenced elsewhere through student ID.

6
Differentiate between centralized and distributed database systems.

A centralized database stores all data in a single location, such as a main server computer. All users access this central repository. This simplifies management but creates a single point of failure.

A distributed database spreads data across multiple physical locations connected through a network. Data may be partitioned or replicated across these locations.

Centralized databases are typically easier to maintain and secure but can become bottlenecks with many users. Distributed databases offer better scalability and fault tolerance but are more complex to administer.

Example

A small business might use a centralized database on their office server. A multinational company would use a distributed database with servers in different countries to serve local offices faster.

7
Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

A Database Administrator (DBA) is a professional responsible for managing, maintaining, and securing databases within an organization. They ensure that databases run efficiently, securely, and are accessible to authorized users.

Responsibilities of DBA

• Database design and implementation
• Performance tuning and optimization
• Backup and recovery planning
• Security management (user roles, permissions)
• Data integrity and consistency checks
• Monitoring database performance and troubleshooting issues

8
Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller related ones and defining relationships between them.

1st Normal Form (1NF)

Ensures that each column contains atomic (indivisible) values and each record is unique.

Example: A table storing multiple phone numbers in a single column violates 1NF.

2nd Normal Form (2NF)

Meets 1NF and removes partial dependencies; all non-key attributes are fully functionally dependent on the primary key.

Example: In a table with composite key (StudentID, CourseID), storing StudentName causes redundancy.

3rd Normal Form (3NF)

Meets 2NF and removes transitive dependencies; non-key attributes should not depend on other non-key attributes.

Example: If Student table stores both StudentID and DepartmentName, and DepartmentName depends on DepartmentID, move it to another table.

9
Explain the terms: primary key, foreign key, and candidate key with examples.

Primary Key: A column (or set of columns) that uniquely identifies each row in a table. It cannot have null or duplicate values.

Foreign Key: A column that creates a relationship between two tables. It refers to the primary key in another table.

Candidate Key: All columns that can uniquely identify rows. One of them is selected as the primary key.

Example

In a Student table, StudentID can be the primary key. Email and RollNumber may be candidate keys. In an Enrollment table, StudentID can be a foreign key referencing the Student table.

10
What is SQL? Explain its components and common functions.

SQL (Structured Query Language) is a standardized language used to interact with relational databases. It is used to create, read, update, and delete data.

Components of SQL

• DDL (Data Definition Language): Defines database schema (e.g., CREATE, ALTER, DROP)
• DML (Data Manipulation Language): Handles data manipulation (e.g., SELECT, INSERT, UPDATE, DELETE)
• DCL (Data Control Language): Controls access (e.g., GRANT, REVOKE)
• TCL (Transaction Control Language): Manages transactions (e.g., COMMIT, ROLLBACK)

Common Functions

• SELECT: Retrieves data from one or more tables
• WHERE: Filters records
• GROUP BY: Groups records with identical data
• JOIN: Combines rows from multiple tables
• ORDER BY: Sorts the results
